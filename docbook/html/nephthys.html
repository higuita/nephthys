1.1.†Nephthys

Nephthys is a management tool for file sharing via usual file transfer methods like WebDAV and FTP. In general Nephthys provides a web interface for handling file sharing slots (so called 'buckets') and the directory structure in the file system. Nephthys it self does not provide any functionallity for WebDAV or FTP - it relys instead of well known applications like Apache, proftpd, ... .

The interface has been kept very simple so that really everybody can use it to share files.

1.2.†Requirements

Nephthys is a written in PHP5. As backend, to store user- and bucket informations, either a MySQL- or SQLite database can be used.

  • Linux-based operating system

  • Apache HTTP webserver (http://httpd.apache.org/)

  • PHP5 (http://www.php.net/)

  • proftpd (http://www.proftpd.org, or any other suiteable FTP server)

  • MySQL (>= 5.0, http://httpd.apache.org)) or SQLite (2.0, http://www.sqlite.net)

Chapter†2.†Definitions and Terms

2.1. Bucket

Nephthys uses some terms in its web interface.

2.1.†Bucket

A bucket (as the name says) is a virtual pot where files and directories can be thrown in or fished out. Usually the bucket has lifetime - when reached, it will be removed automatically including all data within.

3.1.†I want to share some files

The following example describes the procedure the share files with some one else. For this a bucket needs to be created, afterwards it can be filled with data. Then the receiver has to be notified how he can access the data.

To start, open Nephthys in your browser and login.

Figure†3.1.†Nephthys - waiting for login


3.1.1.†Create bucket

First of all, a bucket needs to be created. A bucket gets a name to be identified easily afterwards. So, follow the procedure below:

  1. Click on "I want to share some files".

    Figure†3.2.†Nephthys - Start Page without any exiting buckets


  2. "Give the bucket a name" - enter a name for the bucket.

    Figure†3.3.†Nephthys - Create a Bucket - I want to share some files


  3. "Receivers email" - enter the e-mail address of the receiver. This is field is optional and is used if you want to notify the receiver out of Nephthys. If you leave this field empty and after creating, notify the bucket, you will receive the notification email (to forward it manually for example).

  4. "When will this bucket expire?" - epiration time. Unprivileged users are not permitted to create never expiring buckets. But a privileged user can set it for them.

  5. "Text to be added to notification email" - optionally you can enter a text into the input field, wh—ñch will be included into the notification email.

  6. Click on the save-button to let Nephthys create the bucket for you.

    Figure†3.4.†Nephthys - I want to share some files - Start Page with the newly created bucket


3.1.2.†Upload files

After the bucket has been created, it is time to fill it with some stuff.

3.1.2.1.†WebDAV

On the "Start Page" you will find on the bottom of the screen a list of all your buckets. On of the column contains the link to "WebDAV". If you are using a Microsoft Windows operating system, you can simply click with the left mouse-button on this link. After some seconds, Windows Explorer should appear in an extra window in which you can drag & drop files into. Non-MS users need to use their favorite WebDAV browser - for example Gnome Desktop handles it internally (for example with Nautilus), Apple's Mac OS-X too.

3.1.2.2.†FTP

On the "Start Page" you will find on the bottom of the screen a list of all your buckets. On of the column contains the link to "FTP". The most browsers are capable of browsing FTP directories directly and permitting downloads. Internet Explorer is capable of switching to a "Explorer"-mode where you can drag & drop files. For all others normally you have to use a dedicated FTP client (Filezilla for example) to upload files.

3.1.3.†Sharing

You have now filled your bucket with all the necessary data. Now it is time to share it. There are several ways to do this.

Use the "Notify"-Feature of Nephthys (Start Page -> Locate bucket in list -> Actions -> Notify). It will send an e-mail to the receiver containing necessary details how to access the buckets. If you have specified the receiver's e-mail address when creating the bucket the e-mail will be directly sent to him. If you do not entered the e-mail address, the e-mail will be delivered to your e-mail address instead (that one entered in your profile). You can also copy the WebDAV link and FTP link from the bucket listing and sent it manually.

3.2.†Someone wants to send me files

The following example describes the procedure if some one want to send you files. For this you will need to create a bucket on your own first, afterwards you send the receiver a notification containing links to the bucket so he can upload data to it. Then the receiver notifies the sender, that he has finished uploading.

3.2.1.†Create bucket

First of all, a bucket needs to be created. A bucket gets a name to be identified easily afterwards. So, follow the procedure below:

  1. Click on "Someone wants to send me files".

    Figure†3.5.†Nephthys - Start Page without any exiting buckets


  2. "Give the bucket a name" - enter a name for the bucket.

    Figure†3.6.†Nephthys - Create a Bucket - Someone wants to send me files


  3. "Receivers email" - enter the e-mail address of the receiver.

  4. "When will this bucket expire?" - expiration time. Unprivileged users are not permitted to create never expiring buckets. But a privileged user can set it for them afterwards.

  5. "Text to be added to notification email" - optionally you can enter a text into the input field, wh—ñch will be included into the notification email.

  6. "Send email notification to receiver..." - if this option is checked, immediately after pressing the save button, an e-mail with information on howto access the bucket will be sent to the receiver. Otherwise the notification needs to be invoked manually via the "Start Page".

  7. Click on the save-button to let Nephthys create the bucket for you.

    Figure†3.7.†Nephthys - Someone wants to send me files - Start Page with the newly created bucket


If you have checked "Send email notification to ...", a notification has been sent to the receiver containing information on howto access the bucket.
